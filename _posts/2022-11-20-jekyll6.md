## 1. Paste Image: Path
실제 이미지 파일이 저장될 경로에 대한 옵션입니다. ( 자동으로 폴더 생김 )

미작성시 해당 파일이 있는 위치에 저장됩니다.
<br><br>

  - 현재 위치에 images 폴더를 만들려면?

    ${currentFileDir}/images/

  - 최상단 루트 디렉토리에 images 폴더를 만들려면?

    ${projectRoot}/images/

    위 처럼 설정시 ${projectRoot}/a/b/c/d.md 에서 작업을 하더라도 ${projectRoot}/images/ 에 저장된다.

<br><br>

--- 

## 2. Paste Image: Bass Path
실제 문서 작성 시 타이핑 될 주소 경로.

${projectRoot}/a/b/c/d.md 에서 작성할 경우


${projectRoot} 로 설정시
images/2022-11-20-13-35-15.png

Bass Path의 경우 실제 작성되는 부분이므로, 현재 작성중인 문서의 위치에서 참조를 한다.
그러므로 위에처럼 images/ 가 바로나오면 루트디렉토리가아닌,
현재 파일이있는 디렉토리에서의 images 폴더를 찾게된다.


그러므로 현재 위치에서 참조하기때문에 아래처럼 해야 한다. 
( ../ 이전폴더로 간다는 뜻 )
${currentFileDir}
../../../images/2022-11-20-13-34-50.png

## 사용을 하다보니..
보통 블로그 글 작성시 맥북 기본 캡처를 많이 쓰는데,
캡처한 후 바탕화면에 생긴 파일을 복사 후 마크업에 Paste Image를 이용 붙혀넣기 하면 아래 같은 아이콘이 나온다.
![](../images/2022-11-20-12-33-53.png)


그런데.. 캡처후 파일이 저장되기전 미리보기 상태에서 붙혀넣으면 동작을 한다.
  ![](../images/2022-11-20-12-36-32.png)

  


그런 후에 클립보드에 있는 이미지를 바로 불러오면 사용이된다..
![](../images/2022-11-20-12-35-27.png)


이걸 보니 맥에 저장된 파일은 동작이안되고, 클립보드에 남아 있을때만 저장이 되는 것 같다.
( 버그인 것 같아요.. )

그래서 이렇게 사용하기로 했다.
맥북 부분 캡처 할 때 Shift + Command + control 4로 캡처시 파일저장이아닌
클립보드로만 저장을 바로한고

블로그 글을 쓰면서 바로바로 클립보드에넣어서 붙혀넣기 해서 사용하는 것이 정신건강에 좋을 듯 하다..
