# 메소드
메소드 선언은 선언부와 실행 블록으로 구성된다.
메소드 선언부를 메소드 시그니처(signature)라고 한다.

## 메소드의 구조


    리턴 타입 메소드 이름 (매개변수 선언, ...){

                실행 블록
    }

리턴 타입 : 메소드가 리턴하는 결괏값의 타입을 설정한다.

메소드 이름 : 메소드의 기능이 잘 드러나도록 식별자 규칙에 맞게 이름을 지어야 한다.

매개 변수 선언 : 메소드를 실행할 때 필요한 데이터를 받기 위한 변수를 선언한다.

메소드 실행 블록 : 실행할 코드를 작성한다.


- 리턴 타입
- 리턴 값이 없을 수도 있으며 이때는 void로 표기한다.
- 리턴 값이 있을 경우 해당하는 타입의 변수로 값을 저장할 수 있다.
- 리턴 값이 있어도 실행만 하고 값을 저장하지 않아도 된다.
  


매개변수의 개수를 모를 경우, 혹은 매개변수의 개수를 유연하게 하고 싶은 경우

자바에서는 다음과 같이 사용 가능하다.

``` java
int sumArray(int ... values){
    int sum = 0;
    for(int i = 0; i<values.length; i++){
        sum += values[i];
    }

    return sum;
}
```


## 리턴 문
리턴 값이 있는 메소드라면 반드시 return 문을 사용해서 리턴 값을 지정해 줘야 한다.
return 문이 없다면 컴파일 에러가 발생한다.


리턴문은 리턴 타입이거나 리턴 타입으로 변환될 수 있어야 합니다.
예를 들어 리턴 값이 int인 메소드에 byte, short 값이 리턴되어도 상관없습니다.
byte와 short는 int로 자동 타입 변환 되어 리턴되기 때문입니다.


리턴 값이 없는 메소드의 경우에
return; 을 작성할 경우 메소드 종료의 의미로 사용됩니다.


# 인스턴스 멤버와 정적 멤버


## 인스턴스 멤버
인스턴스 멤버는 객체마다 가지고 있는 멤버를 뜻하고,
정적 멤버는 클래스에 위치시키고 객체들이 공유하는 멤버를 말합니다.

인스턴스 멤버란 인스턴스를 생성한 후 사용할 수 있는 필드와 메소드를 말하는데,
인스턴스 필드와 메소드는 객체에 소속된 멤버이기 때문에 객체 없이는 사용이 불가능합니다.


## 정적 멤버와 static
정적은 고정된이란 의미입니다. 정적 멤버는 클래스에 고정된 멤버로서 객체를 생성하지 않고
사용할 수 있는 필드와 메소드를 의미합니다.

정적 필드와 정적 메소드를 선언하려면 필드와 메소드 선언 시 static 키워드를 붙이면 됩니다.


        public class{

                //정적 필드
                static type field1;

                //정적 메소드
                static returnType method1(){

                }


        }


정적 필드와 메소드는 클래스 로더가 클래스 파일(바이트 코드)를 로딩해서 메소드 메모리
영역에 적재할 때 클래스 별로 관리됩니다.


정적 멤버의 경우 클래스 이름을 사용하여 객체를 생성하지 않아도 바로 사용 가능합니다.

        클래스.필드

        클래스.메소드();

정적 멤버, 메소드의 경우 원칙적으로 클래스 이름으로 접근해야 맞지만, 객체 참조 변수로도 접근 가능합니다.


        Human h1 = new Human();

        h1.필드
        h1.메소드


정적 메소드 선언 시 객체가 없어도 실행되기 때문에,
정적 메소드 내부에 인스턴스 필드나, 인스턴스 메소드를 사용할 수 없고
this 키워드도 사용이 불가합니다.



# 패키지와 접근 제한자.

## 패키지
패키지의 물리적인 형태는 파일 시스템의 폴더(디렉토리)
패키지는 단순히 파일 시스템의 폴더 기능만 하는 것이 아니라 클래스의 일부분으로,

클래스를 유일하게 만들어주는 식별자 역할을 한다.

클래스 이름이 동일하더라도 패키지가 다르면 다른 클래스로 인식한다.


## 접근 제한 자
클래스와 인터페이스를 다른 패키지에서 사용하지 못하게 막거나
필드나 메소드를 사용하지 못하도록 막아야 되는 경우 사용한다.


public : 외부 클래스가 자유롭게 사용 가능하다.
protected : 같은 패키지 또는 자식 클래스에서 사용 가능
private : 외부에서 사용하지 못한다.

위의 설정을 해주지 않으면 default 접근 제한자가 기본으로 적용됩니다.

### 클래스의 접근 제한
defatul : 같은 패키지에서 제약 없이 사용 가능, 다른 패키지에서 사용 불가
public : 같은 패키지 + 다른 패키지에서도 아무런 제약 없이 사용 가능.


### 생성자의 접근 제한
public : 모든 패키지에서 아무런 제한 없이 생성자를 호출 가능.
protected : 같은 패키지, 다른 패키지가 해당 클래스의 자식이라면 생성자 호출 가능
default :   같은 패키지, 다른 패키지는 호출 불가
private : 오로지 클래스 내부에서만 생성자 호출 -> 객체 생성 가능

### 필드와 메소드의 접근 제한
public :  모든 패키지에서 제한 없이 필드 메소드 사용 가능
protected : 같은 패키지, 다른 패키지가 해당 클래스의 자식이라면 사용 가능
default : 같은 패키지 내에서만 사용 가능
private : 클래스 내부에서만 사용 가능



# Getter/Setter
필드를 객체 외부에서 직접적으로 접근하는 것을 막기 위해 사용한다.



예를 들어 사람의 몸무게를 필드값으로 쓰는 경우
사람의 몸무게는 음수가 될 수 없으므로, 직접 필드에 접근한다면 음수로 설정이 가능할 것이다.


외부에서 데이터를 읽을 때에도 getter를 사용하여
데이터를 가공하여 잘 전달해 주는 것이 좋다.

필드의 타입이 boolean형 일 경우 getter은 get.. 이아니라 is로 시작하는 게 관례이다.

ex)

    private boolean stop;

    //Getter
    public boolean isStop(){
        return stop;
    }

    //Setter
    public void setStop(boolean stop){
        this.stop = stop;
    }


크게 2가지의 장점이 있는데

1. 데이터의 객체의 무결성을 지킬 수 있다. (몸무게 -> 음수 불가)
2. 필드값을 private로 두어서 외부에서 바로 필드로 접근하지 않고
메소드를 거쳐오기 때문에 함부로 접근할 수 없다. (메소드 내부에서 보안성을 높일 수 있으니)
