# 인증/인가
사실 스프링에서 인증, 인가와 관련되어 스프링 시큐리티가 관리해 준다고 한다.  

하지만 해당 개념이 잡히기 전에 바로 스프링 시큐리티를 사용하는 건 너무  
어렵기만 하고 재미가 없기 때문에 차근차근 스프링 시큐리티 없이
공부하여 구현부터 해보았다.

## 인증(Authentication)
해당 유저가 실제 유저인지 인증하는 개념이다.  
스마트폰에 지문인식, 이용하는 사이트에 로그인 등과 같이, 실제 그 유저가 맞는지를 확인하는 절차이다.


## 인가(Authorization)
해당 유저가 특정 리소스에 접근이 가능한지 허가를 확인하는 개념이다.  
예를 들어 관리자 페이지-관리자 권한 같은 것들!

<hr>

웹 애플리케이션은 기본적으로  
비 연결성(Connectionless) 무상태(Stateless)로 이루어져 있기 때문에

서버는 클라이언트와 연결되어 있지 않고 클라이언트의 상태를 저장하지 않는다.  

그렇기 때문에,  
로그인을 하면 로그인이 계속된 것처럼 해주어야 하고,  
비연결성이 연결성인 것처럼 보이게 해줘야 되고 무상태가 상태인 것처럼 해줘야 한다.

이것들을 쿠키, 세션 등으로 구현할 수 있지만  
이번에는 JWT로 구현해 보았다.

## 느낀 점
이렇게 실제 데이터베이스 서버에 연결하고,  
EC2 서버에 호스팅 하는 과정들을 진행해 보니,

생각보다는 과정이 쉽다고 느꼈지만, 또 아래와 같은 것들도 느꼈다.
 
- 서버 보안의 중요성

- HTTP 공부
- IP 개념

실제 서버를 호스팅 하는 과정이 쉬워진 만큼 더욱 신경 써야 할 부분이 많지 않을까?

# TIL, 글쓰기
블로그 포스팅 같은 경우 현재 TIL도 그렇고  
Java, Spring 같은 경우도 그렇고 중요한 내용의 경우 

가독성이 없고 중요한 내용이긴 하지만 너무 글의 양이 많다.  
이제부터는 깔끔하게 중요한 내용만 가독성 있게 정리하여 포스팅하는 연습을 해야 할 것 같다.

그리고 TIL은 한눈에 성장과정을 볼 수 있도록 제목에 최대한 배운 것의 중요 키워드를 넣어야 할 것 같다.

또한 배운 내용 들을 정리하는 과정에서 해당 카테고리에 포스팅하고, 또
TIL(WIL)에 포스팅하니 한눈에 정리도 안되고, 힘들기도 하고 여러 글을 참조해야 하는 느낌이 들어서

TIL(WIL)에는 배운 내용들을 내 포스팅 링크로 걸고  
해결한 점, 몰랐던 점, 배운 점 등만 기술해야겠다.
