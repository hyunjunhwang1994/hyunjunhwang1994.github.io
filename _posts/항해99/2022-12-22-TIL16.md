## 알고리즘 주 차에서 힘들었던 점

솔직히 처음에는 너무 어려웠다.

별찍기도 힘들고, 별을 왜 찍어야 하나라는 생각도들고...ㅋㅋ

짧으면 30분 길면 1시간 정도 투자를 해보고 구현하기가 힘들면
문제의 정답을 보고 이해해서 내 것으로 만드는 과정부터 시작하였다.

왜냐하면 시간 자체도 일주일 정도로 한정되어 있기 때문에,  
정말 알고리즘 공부를 처음 해보는 경우 이 문제들을 처음부터 실력으로

## Java


내일 금요일 자바의 객체지향을 주제로 한 자바 스터디 모임이 있다.

그렇기도 하고, 스프링을 공부하기 전 자바의 객채지향(클래스, 인터페이스 등)에 대해서 완벽하게 익히고자

혼자 공부하는 자바라는 책을 6챕터부터 한 번 더 필독하였다.

오늘 공부한 내용 중 간단하게 정리해 보자면

- 객체 지향 프로그래밍
객체는 속성(필드)과 동작(메소드)으로 구성되어 있음,

객체관의 관계 -> 집합 관계, 사용 관계, 상속 관계를 사용하여  
현실 세계의 객체를 소프트웨어 객체로 설계 (객체 모델링)하여 구현하는 프로그래밍 방법

클래스(객체)의 구성 맴버  
필드, 생성자, 메소드

- 필드
일부 사람들은 클래스 멤버 변수라고 부르기도 하는데,
될 수 있으면 필드라는 용어를 그대로 사용하는 게 좋다.

- 필드는 초기화를 해주지 않고 생성 시 알아서 초깃값이 기본값으로 들어간다.
- 필드를 사용한다는 것? 필드값을 읽고 변경한다는 것

- 생성자
- new 연산자로 호출되는 중괄호{}안 블록.
- 객체 생성 시 초기화를 담당한다.
- 모든 클래스에 반드시 하나 이상 존재
- 객체 초기화란 필드를 초기화하거나 메소드를 호출해서 객체를 사용할 준비를 하는 것을 말한다.
- new 연산자에 의해 생성자가 성공적으로 실행되면 Heap 영역에 객체가 생성되고 객체의 번지가 리턴된다.
- 모든 클래스는 생성자가 반드시 존재하며, 하나 이상 가질 수 있고
- 명시적으로 선언한 생성자가 1개라도 있을 경우 컴파일러는 기본 생성자를 추가하지 않음.
- 생성자 간 오버 로딩이 가능 (매개변수의 타입과 개수 그리고 선언된 순서가 달라야지만 가능)

더 자세한 내용은 따로 포스팅으로 정리해 보려고 한다.
