## 오늘의 배움 포인트 정리.

### 상수
상수는 일반적으로 불변의 값을 상수(constant)라고 한다.
자바에서의 상수는 final이 아닌 static final으로 사용한다.

왜냐하면, 불변의 값은 객체마다 저장할 필요가 없는 공용성을 띠고 있고,
여러 가지 값으로 초기화될 수 없기 때문에 static final로 사용한다.
<hr>

### 자동 타입 변환
자동 타입 변환은 부모 타입 변수 = 자식 타입으로 가능하다.

```java
Cat cat = new Cat();
Animal animal = cat;
```

이때 변수 cat, animal은 타입만 다를 뿐 동일한 Cat 객체를 참조한다.  
부모 타입으로 자동 타입 변환되면, 부모 클래스에 선언된 필드와 메소드만 접근이 가능하다.

변수는 자식 객체를 참조하지만, 변수로 접근 가능한 멤버는 부모 클래스 멤버로만 한정된다.

그러나 예외가 있는데, 부모 객체의 메소드가 자식 클래스에서 재정의되었다면 자식 클래스의 메소드가 대신 호출된다.
(다형성)

그냥 자식 타입으로 사용하면 될 것을 자동 타입 변환이 왜 필요할까?

-> 다형성을 구현하기 위해서. 필드의 타입을 부모 타입으로 선언하면 다양한 자식 객체들이 저장될 수 있기 때문에
