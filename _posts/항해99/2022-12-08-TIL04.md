# 항해 99 입학시험

오늘은 대망의 입학시험을 보는 날이다.    
시험 문제는 웹 개발 종합반 강의를 어느 정도 숙달했다면 충분히 쉽게 나왔다.

사실 전날 CRUD 중 Delete 과정을 미리 연습해 보았는데,  
Update가 나와서 코드만 살짝 수정 후 테스트하고 바로 제출해서 한 30분 걸린 것 같다!

강의를 많이 듣지 않았거나 뼈대를 미리 만들어 놓지 않았다면 
살짝 어려웠을 수도 있을 것 같은 수준의 난이도인 것 같다.

사실 이런 입학시험 가지고 탈락 시킬 것 같진 않다.  
정말 시험을 안 보거나, 의지가 없는 사람만 골라내려고 하는 것 같다.

# 지금까지 돌아보자면.

항해를 시작하고 웹 개발 강의를 처음 받았을 때는, 솔직히  
파이썬과 플라스크 기반에 NoSQL인 mongo DB로 강의를 하기에,

자바는 언제 배우고, 스프링은 언제 배우지?라는 생각을 했다.

하지만 아직 초반이고, 플라스크를 썼기에 어느 정도 웹 사이클을 빨리 경험한 것뿐만 아니라  
빨리 플라스크의 구조를 조금이라도 익혔기에

웹에 있어 중요한 기능들 예를 들면 세션, 암호화, MVC 구조, JWT 등을  
파보려고 하고, 어느 정도는 파본 것 같아 지금에 와서는 만족하고 있다.


이러한 경험을 토대로 다른 언어, 프레임워크 들도 파고들 수 있는 실력을 기른 것 같아 뿌듯해졌다.  
앞으로 포스팅은 계속하는 걸로... 개념 정리에 있어서 정말 도움이 되는 것 같다.

글을 쓰고 정리하면서, 더 검색해 보고 문서를 읽으면서, 정말 추상적으로만 그려졌던 개념이  
어느 정도는 확실하게 그려지는 경험을 짧은 시간에 많이 해 본 것 같아 기분이 좋다.  



## 앞으로 어떻게 해야 할까?
입학시험을 보고 리액트, 스프링, 노드 js 3가지 중의 주특기를 골라야 했다.  
나는 애초에 스프링을 배우고 싶기에 스프링을 골랐고 후회하지 않는다.

앞으로도 지금처럼 모르는 것이 나왔을 때 대충 넘기지 않고  
정리하면서 글로써 남기는 습관을 들이면 좋을 것 같고 이제부터는 자바 공부도 좀 많이 해야 할 것 같다!!

node.js , react, spring 시니어 개발자 님들의 주특기 세션을 들어본 결과  
결국 회사에서는 당장 실력이 조금 떨어지더라도 (물론 기본은..?) 더욱 발전할 수 있는 사람과

커뮤니케이션을 잘할 수 있는 사람을 원하는 것 같았다.
